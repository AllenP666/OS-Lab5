# Лабораторная работа 5: Сетевой мониторинг температуры (SQLite + HTTP)

В данном проекте расширен функционал логгера из 4-й лабораторной работы. Текстовые логи заменены на базу данных SQLite. Добавлен HTTP-сервер на сокетах для публикации данных и веб-клиент для отображения графиков.

## Сборка проекта
- **Windows:** запустить `update_build.cmd`
- **Linux:** дать права на выполнение и запустить `./update_build.sh`

## Запуск
Для быстрой проверки можно использовать скрипт автоматического запуска (запускает симулятор и сервер):
- Windows: `run_all.cmd`
- Linux: `./run_all.sh`

Либо запустить вручную из папки `build` в двух разных терминалах:
1. `./sensor_simulator` (генерирует данные)
2. `./weather_server virtual_com` (читает данные и поднимает сервер)

## Использование
После запуска сервер доступен по адресу:
`http://localhost:8080`

В веб-интерфейсе отображается текущая температура и график истории. Можно выбирать период отображения (минута, час, день).

**API эндпоинты:**
- `GET /api/current` — получить последнее измерение в JSON.
- `GET /api/history?seconds=N` — получить массив измерений за последние N секунд.

## Файлы
- `weather.db` — файл базы данных SQLite (создается автоматически в папке build).
- `index.html` — клиентская часть (копируется в build при сборке).
